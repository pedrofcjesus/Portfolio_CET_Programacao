# Web Development Exercises (Frontend & Backend)

This folder contains practical exercises and small projects focused on web development. The exercises here demonstrate a progression from static page design to dynamic, database-driven web applications.

---

## 🚀 Key Technologies & Competencies

* **Frontend Development:** Structuring web pages with **HTML5**, styling and responsive design with **CSS3**, and adding interactivity using **JavaScript** (DOM manipulation and asynchronous operations).
* **Backend & Server-Side Logic:** Utilizing **PHP** to handle server-side processing, form validations, and session management.
* **Database Integration:** 
  * **Relational Databases (SQL):** Connecting web applications to MySQL databases for structured data storage and retrieval.
  * **NoSQL Databases:** Integrating cloud-based NoSQL solutions, such as **Firebase**, for real-time data management.

---

## 📁 Folder Overview

Instead of a strict exercise-by-exercise breakdown, this repository is organized by project or conceptual topic. Inside, you will find:
* Static web pages and UI components.
* Interactive client-side scripts.
* Full-stack implementations integrating PHP and SQL/NoSQL databases.

---

## 🛠️ How to Run

Since these exercises span both frontend and backend technologies, the execution method depends on the specific project:

1. **Frontend-Only Exercises (HTML/CSS/JS):**
   * Simply open the `index.html` file directly in any modern web browser.
   
2. **Backend/Full-Stack Exercises (PHP & SQL):**
   * These projects require a local server environment (such as XAMPP, WAMP, or MAMP).
   * Place the project folder inside the local server's document root (e.g., the `htdocs` folder in XAMPP).
   * Ensure your local MySQL/MariaDB server is running.
   * Open your browser and navigate to `http://localhost/folder_name`.
> 💡 **Note on Cloud Services & Security:** For exercises utilizing Firebase, the API keys have been intentionally removed from the source code for security reasons. To ensure the projects remain functional and easy to evaluate, I have implemented a JavaScript fallback function that handles data locally. This allows the application to be viewed and tested immediately without requiring external database connections or manual API configuration.
